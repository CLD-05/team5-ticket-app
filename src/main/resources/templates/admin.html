티켓웨이브 관리 가이드

본 관리용 콘솔은 티켓팅 시연 및 데이터 통제를 위한 도구입니다. 아래 기능을 지원합니다.

💡 시연용 영구 고정 상태(Permanent Mode) 안내:
  • 시연 과정에서 시간이 흘러도 예매 상태가 변하지 않도록, 시뮬레이터의 간편 버튼 적용 시 현재 시각 대비 10년 단위의 미래/과거 타임스탬프를 할당합니다.
  • 예매 오픈 전 카운트다운: '공연 & 시뮬레이터' 탭에서 대상 공연 선택 후 [영구 예매 대기 설정] 버튼을 누르면 오픈 시간이 10년 뒤로 밀려, 영구적인 카운트다운 상태를 안전하게 시연할 수 있습니다.
  • 상시 예매 및 대기열: [영구 예매 진행 설정]을 클릭하면 오픈은 10년 전, 마감은 10년 뒤로 세팅되어 영구적으로 예매가 활성화된 상태가 유지됩니다. 대기열 및 좌석 예약 흐름을 시연하십시오.
  • 상시 예매 마감: [영구 예매 마감 설정]을 누르면 마감 시간이 수년 전으로 고정되어, 언제 접속하더라도 예매 마감 안내가 영구적으로 노출됩니다.

공연 관리

시스템에 예매가 가능한 새로운 공연을 등록하거나 기존 공연을 영구 삭제합니다.

⚠️ 등록 공연 삭제

선택한 공연과 그에 연관된 모든 좌석, 등급 및 회원 예매 정보가 DB에서 완전히 삭제됩니다.

📸 공연 포스터 이미지 등록

선택한 공연의 포스터 이미지를 업로드하여 등록합니다. (S3 연동)

대기열 관리 및 리셋

고부하 상황 복구용 Redis 캐싱 대기열 초기화 도구입니다.

💡 대기열 리셋 기능이란? 실시간 대규모 접속 시, 사용자들은 먼저 Redis Sorted Set 구조의 대기열에 진입하고 차례대로 입장 권한 토큰을 발급받아 좌석 선택 화면에 도달합니다.
  • 공연별 리셋: 특정 공연의 대기 큐 및 이미 발급된 입장 토큰만 선별적으로 삭제하여 진입로를 복구합니다.
  • 전체 강제 리셋: Redis 내의 모든 대기열 관련 데이터(`queue:*` 와 토큰 키)를 삭제하여 무임승차를 차단하고 초기 클린 상태로 원복합니다.
